Detailed Description

Finds cheapest paths in a graph from a set of source nodes to all nodes, or a negative cycle reachable from the sources.

Note
Edges count as 2-cycles.

There is no restriction on the cost function (as opposed to AStar and Dijkstra), but if a negative cycle is reachable from the sources, the algorithm terminates and does not calculate the distances.

If the cost function is non-negative, use Dijkstra, as it runs faster.

Querying the results:

  • If a negative cycle has been reached, then NegativeCycle is not null and contains such a cycle.
    • In this case, GetDistance, GetParentArc and GetPath throw an exception.
  • If no negative cycle could be reached, then NegativeCycle is null.
    • In this case, use GetDistance, GetParentArc and GetPath for querying the results.
    • For unreachable nodes, GetDistance, GetParentArc and GetPath return double.PositiveInfinity, Arc.Invalid and null respectively.
